# Phase 02 Plan 01: Retrieval Models And Normalization Foundation

## Objective

Define the retrieval envelope and normalization rules that every later retrieval call will use, keeping transport, filtering, and deduplication behavior testable outside the UI layer.

## Must Haves

- A structured retrieval envelope separates normalized `places`, search-level `metadata`, and structured `error`
- Normalized place records preserve the required map/ranking fields plus the agreed optional fields
- Duplicate handling is deterministic and keyed strictly on `place_id`
- Non-operational businesses are removed during retrieval
