# Phase 02 Plan 02: Geocoding And Places Retrieval Service

## Objective

Build the geocode-first retrieval service that accepts the Phase 1 `SearchQuery`, resolves the city, fetches up to three pages of category-based place results within the requested radius, and returns a structured result envelope for later ranking and UI phases.

## Must Haves

- Retrieval consumes `SearchQuery` instead of reading raw widget values
- City geocoding happens before place retrieval
- Places fetching honors a three-page cap and returns partial results with warnings on later-page failures
- The service returns normalized `places`, separate `metadata`, and structured `error` behavior for geocoding failures and empty results
