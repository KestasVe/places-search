# Phase 01 Plan 02: Search Shell UI Wiring

## Objective

Implement the approved Phase 1 interface in `app.py` so users can enter city, category, and radius through a focused Streamlit shell and produce a stable normalized query object on explicit submit.

## Must Haves

- The UI follows `01-UI-SPEC.md` for form order, copy, colors, and interaction rules
- Search only triggers on the `Search places` button
- The button is disabled until required fields are non-empty after light normalization
- Inline validation is shown reactively at the relevant fields while required inputs are empty, and the resulting normalized query object is available for later phases
