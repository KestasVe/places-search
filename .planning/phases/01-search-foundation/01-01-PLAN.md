# Phase 01 Plan 01: Search Query Foundation

## Objective

Create a small, testable query boundary for the search form so later phases consume a stable normalized object instead of raw widget state.

## Must Haves

- A helper module defines the six-field normalized query object exactly as agreed in `01-CONTEXT.md`
- Normalization trims outer whitespace, collapses repeated internal spaces, and lowercases normalized text values
- Validation identifies missing city and category inputs before search execution
- The foundation logic remains independent of Streamlit APIs so it can be unit tested
