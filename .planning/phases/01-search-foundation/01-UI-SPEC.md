| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Validation and helper spacing inside a control group |

| lg | 24px | Section padding within the search shell |
| xl | 32px | Gap between intro copy and the form/results boundary |
| 2xl | 48px | Major separation between page header and body content |

## Layout Contract

- Phase 1 uses a single-page Streamlit shell with a strong page title, a short explanatory caption, and one clearly grouped search area.
- The search area should read top-to-bottom in this order: city input, category input, radius control, then primary action.
- Use one dominant search container surface rather than scattering controls across unrelated sections.
- Keep the form narrow enough for fast scanning on desktop, while remaining usable in Streamlit's default responsive layout.
- Results tables, maps, and ranking visualizations are out of scope in this phase; only the query shell and validation states belong here.

## Interaction Contract

- Search runs only on explicit button click.
- The primary button stays visually disabled until city and category are non-empty after light normalization.
- Validation appears inline at the relevant field group, not as a generic page-level error.
- City input label or helper text must explicitly signal that Lithuanian and English city names are both acceptable.
- Category input uses placeholder examples only: `Kebabai`, `Museums`, `Cafes`.
- Radius uses a slider with a visible 1-50 km range and a default of 10 km.

## Query State Contract

- The UI hands off a normalized query object with exactly these fields:
  - `city_raw`
  - `city_normalized`
  - `category_raw`
  - `category_normalized`
  - `radius_km`
  - `radius_m`
- Raw values remain suitable for display back to the user.
- Normalized text trims outer whitespace, collapses repeated internal spaces, and lowercases for internal use.
- Phase 1 does not expose geocoding, coordinates, API metadata, or caching controls in the interface.

## Visual Tone

- The interface should feel practical and trustworthy rather than playful or map-heavy.
- Use warm neutral surfaces and one earthy accent to imply place discovery without leaning on tourism cliches.
- Avoid over-designed hero sections, glassmorphism, gradients, or dashboard density in this phase.
- The search shell should feel like a focused tool, not a content site.
